从文本到测试用例

AI 模型回复的是自然语言文本，但我们需要的是结构化的测试用例。这个转换过程就像从一堆散装零件中，按照图纸组装出精密的机械。

掌握这个过程，能帮你在处理任何 AI 输出时，都知道如何提取有用的信息。

💡 为什么要关心文本解析？
AI 的输出往往是"接近正确"的文本，如何从中稳定提取结构化数据，决定了你的 AI 应用的可靠性。这是所有 AI 应用的核心挑战。

AI 输出的典型格式

系统要求 AI 按照特定格式回复，但实际输出往往有细微差异。

正则表达式的威力

系统使用正则表达式来从文本中提取结构化信息。

解析规则拆解：

^[•*-]\s*用例：(?P<name>.+?)\s*-\s*操作：(?P<operation>.+?)\s*-\s*预期：(?P<expected>.+?)$
^[•*-]\s*
行开头的列表符号 + 可选空格
用例：(?P<name>.+?)\s*
"用例："后面的内容保存为 name
-\s*操作：(?P<operation>.+?)\s*
"-操作："后面的内容保存为 operation
-\s*预期：(?P<expected>.+?)$
"-预期："后面的内容保存为 expected

匹配演示：

输入文本： 
• 用例：登录成功 - 操作：输入正确用户名和密码点击登录 - 预期：跳转到主页面显示欢迎信息
提取结果：
name: "登录成功"
operation: "输入正确用户名和密码点击登录"
expected: "跳转到主页面显示欢迎信息"

版本前缀注入

根据 output.version_prefix 配置，自动为每条用例的模块路径加上版本号

版本前缀处理代码：

version_prefix = self.config.output.version_prefix

for case in cases:
    module = (case.module or "").strip()
    if module.startswith("/"):
        # "/登录" → "/v1.3.7/登录"
        case.module = f"/{version_prefix}{module}"
    elif module:
        # "登录" → "/v1.3.7/登录"
        case.module = f"/{version_prefix}/{module}"
    else:
        # 空模块 → "/v1.3.7"
        case.module = f"/{version_prefix}"

为什么需要版本前缀：

  • 多迭代管理：v1.3.7 和 v2.0.0 的用例可以在同一个测试平台里区分
  • 自动拼接：AI 生成的模块路径不需要手动加版本，配置一次全部自动处理
  • 格式容错：无论 AI 输出的路径有没有前导斜杠，代码都能正确处理
LLM_CONVERT_REGEX = re.compile(
    r"^[•*-]\s*用例：(?P<name>.+?)\s*-\s*操作：(?P<operation>.+?)\s*-\s*预期：(?P<expected>.+?)$",
    re.MULTILINE
)

def parse_test_cases(text: str) -> List[TestCase]:
    cases = []
    for match in LLM_CONVERT_REGEX.finditer(text):
        case = TestCase(
            name=match.group('name').strip(),
            operation=match.group('operation').strip(),
            expected=match.group('expected').strip()
        )
        cases.append(case)
    return cases

关键设计考量：

  • 容错性：\s* 匹配可选空格，应对格式差异
  • 贪婪控制：.+? 非贪婪匹配，避免跨字段错误
  • 清理：strip() 去除首尾空格
  • 多行模式：re.MULTILINE 支持每行独立匹配

常见解析失败原因：

1. 格式偏离：AI 没有严格按照要求的格式输出
2. 多行用例：单个测试用例跨越多行
3. 字段缺失：缺少"操作"或"预期"部分
4. 非用例内容：AI 添加了说明文字或总结

提升解析成功率的方法：

  • 优化提示词：在提示词中强调格式的重要性
  • 增加示例：给 AI 提供标准格式的样例
  • 扩展正则：支持更多格式变体
  • 后处理：对解析失败的行进行二次处理
